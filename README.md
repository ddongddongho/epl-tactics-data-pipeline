# EPL 과거 시즌 데이터를 활용한 팀 승리 요인 분석 빅데이터 파이프라인

## 1. 문제 정의
* **해결하고자 하는 문제:** 잉글랜드 프리미어리그(EPL)의 과거 시즌 경기 결과와 팀별 세부 스탯 데이터를 분석하여, 특정 지표(볼 점유율, 유효 슈팅 등)와 경기 승리 간의 상관관계를 도출하고자 합니다. 이를 위해 하둡 생태계(Hadoop Ecosystem)를 활용하여 대용량 CSV 데이터를 처리하고 분석하는 데이터 웨어하우징 파이프라인을 구축합니다.
* **사용할 데이터:** Kaggle(캐글)에서 제공하는 EPL 과거 시즌 경기 결과 및 팀 스탯 CSV 데이터셋

## 2. 기술 스택
본 프로젝트는 수업 실습 환경인 HDP Sandbox(Ambari) 환경 위에서 진행되며, 데이터 수집부터 분석까지 다음의 하둡 에코시스템을 활용합니다.

| 구분 | 사용 도구 | 활용 목적 |
| :--- | :--- | :--- |
| **데이터 저장** | HDFS | 원천 데이터(Raw CSV) 및 중간 정제 데이터의 분산 저장 |
| **데이터 전처리** | Apache Pig | `LOAD`, `FILTER`, `FOREACH` 문을 활용해 원천 데이터에서 결측치를 제거하고 분석에 필요한 특정 칼럼만 추출하여 HDFS에 재저장 |
| **커스텀 데이터 처리** | Python MapReduce (`mrjob`) | 필요 시 파이썬 기반의 MapReduce 코드를 작성하여 팀별 득점 수치나 특정 이벤트 빈도를 집계 |
| **데이터 웨어하우스 & 분석** | Apache Hive | Pig로 전처리된 데이터를 기반으로 Hive 테이블을 생성하고, HiveQL(SQL)을 사용해 `JOIN` 및 `GROUP BY` 연산을 수행하여 최종 인사이트(상관관계) 분석 |

## 3. 구현 계획 (데이터 파이프라인)
1. **데이터 수집 및 적재:** Kaggle에서 EPL 시즌별 경기 통계 데이터(CSV)를 로컬로 다운로드한 후, Ambari Files View 또는 CLI를 통해 HDFS의 `/user/maria_dev/...` 디렉토리에 업로드합니다.
2. **데이터 정제 (Pig):** Apache Pig 스크립트를 작성하여 HDFS에 적재된 CSV 데이터를 읽어옵니다. 분석에 불필요한 칼럼을 제거하고, 데이터를 규격화한 후 결과를 HDFS의 새로운 디렉토리에 저장합니다.
3. **데이터 구조화 (Hive):** Hive에서 `CREATE TABLE` 문을 사용해 스키마를 정의하고, Pig가 출력한 정제된 데이터를 `LOAD DATA INPATH` 명령어를 통해 Hive 테이블에 적재합니다.
4. **데이터 분석 (HiveQL):** 데이터 분석 스튜디오(Data Analytics Studio)나 Beeline 커맨드라인을 통해 Hive SQL 쿼리를 작성합니다. 팀별 승률, 평균 유효 슈팅 수 등을 집계하고 승리 요인을 도출합니다.
5. **결과 시각화 (선택):** Hive에서 쿼리된 최종 결과 데이터를 로컬로 가져와 Python(Matplotlib, Seaborn)이나 엑셀을 활용해 간단한 그래프로 시각화하여 결론을 도출합니다.
